## 5. Industrial Data Strategy

Databricks provides comprehensive industrial data capabilities purpose-built for energy and utilities operations, addressing the unique requirements of SCADA systems, IoT sensor streams, and time-series data that underpin AGL's generation assets, transmission networks, and distributed energy resources.

**SCADA Integration and Operational Technology Connectivity**

Databricks supports direct integration with AGL's SCADA environment through multiple connectivity patterns. The Zerobus Ingest architecture enables real-time data flow from Ignition SCADA and HiveMQ MQTT brokers, forwarding RabbitMQ messages to Azure Event Hubs or AWS Kinesis for immediate processing. This eliminates the traditional air gap between operational technology and IT systems, enabling unified analytics across generation control systems, substation automation, and customer operations.

For legacy SCADA historians including OSIsoft PI System, Wonderware, and GE Proficy, Lakehouse Federation provides read-only federated queries via JDBC connections. Push-down query optimisation executes filters and aggregations at the source system, minimising data movement whilst Unity Catalog enforces consistent access controls. This enables ad-hoc analysis of historical SCADA data without replication, supporting root cause analysis and asset performance investigations.

Structured Streaming processes SCADA telemetry in real-time with exactly-once processing guarantees. Stateful stream processing handles de-duplication of redundant sensor readings, anomaly detection for out-of-range values, and smoothing algorithms for noisy measurements. Watermarking manages late-arriving data from distributed assets with network latency, ensuring accurate aggregations despite clock skew and communication delays.

**IoT Sensor Stream Processing at Scale**

Databricks ingests IoT sensor streams from AGL's distributed energy resources through native cloud integrations. Ignition Gateway and AWS IoT Core connect directly to Event Hubs and Kinesis, whilst Lakeflow Connect provides zero-code ingestion from MQTT brokers, Kafka clusters, and industrial protocols including Modbus and OPC-UA.

For AGL's battery storage facilities and solar farms, Structured Streaming processes millions of sensor readings per second with sub-second latency. Auto Loader incrementally ingests new files from edge gateways writing to ADLS Gen2 or S3, automatically inferring schema changes and handling corrupt records. This supports both real-time monitoring and historical analysis on a single copy of data.

Delta Lake provides ACID transactions for concurrent writes from thousands of distributed sensors, ensuring data consistency despite network partitions and device failures. Z-ordering and liquid clustering optimise query performance for common access patterns including time-range filters, asset-specific queries, and sensor-type aggregations—delivering sub-second response times for operational dashboards.

**Time-Series Data Ingestion and Storage Optimisation**

Databricks optimises time-series data storage through Delta Lake's advanced features. Partition pruning eliminates unnecessary data scanning by organising tables by timestamp hierarchies (year/month/day/hour), whilst Z-ordering co-locates related measurements for 10x faster queries on asset ID and sensor type dimensions.

Predictive Optimisation automatically compacts small files generated by high-frequency sensor writes, reducing metadata overhead and improving query performance. The system analyses access patterns to schedule OPTIMIZE operations during low-usage periods, maintaining 70% faster queries without manual intervention.

For AGL's interval meter data and SCADA telemetry, Delta Lake's time travel provides point-in-time queries supporting regulatory audits and incident investigation. Retention policies balance storage costs against compliance requirements, with configurable retention from 7 to 90 days for deleted files whilst maintaining infinite retention for committed data versions.

**Real-Time Processing and Analytics**

Structured Streaming with Real-Time Mode delivers 40-300ms p99 latency for time-critical applications including frequency regulation, voltage control, and fault detection. This enables AGL to deploy real-time analytics for grid operations where traditional batch processing introduces unacceptable delays.

Lakeflow Spark Declarative Pipelines provide declarative stream processing with automatic state management, checkpointing, and failure recovery. Tumbling and sliding windows aggregate sensor readings over configurable time intervals, whilst stateful joins correlate events across multiple data streams—enabling complex event processing for asset health monitoring and predictive maintenance.

For AGL's battery dispatch optimisation, streaming ML inference applies trained models to real-time market signals and asset telemetry, generating dispatch instructions within seconds of price changes. MLflow Model Serving provides sub-100ms inference latency with automatic scaling from zero to thousands of requests per second.

**Industrial Analytics and Visualisation**

Databricks SQL provides interactive analytics over industrial time-series data with sub-second query response times. Photon engine accelerates aggregations, window functions, and time-series operations common in operational analytics, whilst intelligent caching eliminates redundant computation for frequently accessed dashboards.
