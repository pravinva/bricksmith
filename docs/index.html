1) The Problem

The EDP supports multiple semantic models with different freshness expectations, but refresh orchestration is inconsistent, coupled, and hard to govern at scale.

  • Inconsistent refresh behavior across models and business domains.
  • Shared workflow coupling where one model failure can delay others.
  • No single metadata control plane for SLA policy and trigger behavior.
  • Limited visibility into refresh success, latency, and cost by model.
Trust RiskUnclear dashboard currency for business users
Operational NoiseManual triage and repeated status checks
Cost InefficiencyRefresh patterns not aligned to table change behavior
Scale RiskModel onboarding requires repeat orchestration design

2) The Solution

Introduce a metadata-driven semantic refresh framework on Databricks with model-level isolation, native Power BI tasks, table update triggers for NRT, and full cost/SLA observability.

Model-level refresh Model-based workflow isolation SLA + cost telemetry built-in

Outcome: predictable freshness, lower failure blast radius, better governance posture, and clear operational ownership per model.

3) High-Level Components

A. Metadata & Governance

  • Schema: edp_metadata.semantic_refresh
  • Core tables: semantic_models, table_refresh_config, refresh_execution_log, refresh_cost_tracking
  • SLA policy as data: sla_threshold_minutes, sla_tier

B. Workflow Orchestration

  • One workflow per model per cadence (for fault isolation).
  • Task chain: log start → refresh model → log complete → capture costs.
  • Notifications scoped to model owners and platform support.

 

C. Trigger & Execution Layer

  • Scheduled cadence for daily and medium-frequency refresh.
  • Native table_update triggers for NRT tables with debounce.
  • Databricks Power BI task executes model refresh via connection_name and dataset_id.

D. Monitoring & Cost Intelligence

  • 7-day success rate, SLA breaches, staleness, and duration metrics.
  • Cost per refresh and annualized projection from execution logs.
  • Databricks SQL health scorecard for executive and operational views.

4) High-Level Picture

  1. Gold tables in Unity Catalog emit freshness signals.
  2. Metadata tables define model policy, cadence, trigger mode, and SLA thresholds.
  3. Workflow generator creates model-isolated Databricks Jobs.
  4. Jobs run Power BI refresh tasks and log execution + cost telemetry.
  5. SQL dashboards provide health, SLA, and cost visibility.
Conclusion: Fulton Hogan gets a consistent, auditable, and scalable refresh operating model that improves trust in reporting, isolates failures, and provides better cost control.
